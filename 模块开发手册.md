# 零核服务器模块开发手册

## 概述

零核服务器支持模块化扩展，允许开发者创建自定义功能模块。模块运行在独立的沙盒环境中，支持多视图引擎，确保系统的安全性和稳定性。

## 模块系统特性

### 1. 沙盒隔离
- 每个模块在独立的 VM 沙盒中运行
- 限制模块对系统资源的访问
- 防止模块间相互干扰
- 提供安全的执行环境

### 2. 多视图引擎支持
- 支持 EJS、Handlebars、Pug、Mustache 等多种视图引擎
- 开发者可在配置文件中指定视图引擎
- 自动适配不同引擎的语法

### 3. 受限的模块访问
- 仅允许访问安全的 Node.js 内置模块
- 禁止访问文件系统敏感路径
- 限制网络请求能力

### 4. 模块收藏功能
- 用户可以收藏喜欢的模块
- 显示模块收藏数量
- 用户可查看所有收藏的模块

### 5. 模块评论功能
- 用户可以在模块卡片上发表评论
- 支持回复其他用户的评论
- 支持删除自己的评论（管理员可删除任何评论）
- 评论数据存储在 `./module/data/comments.json`

## 模块配置文件

每个模块都需要一个 `setting.json` 配置文件，格式如下：

```json
{
  "name": "模块名",
  "version": "版本号",
  "description": "模块描述",
  "intro": "简介 (支持 MD, \\n 表示换行)",
  "author": "作者信息",
  "page": true/false,  // 是否拥有页面 (true=UI 模块，false=API 模块)
  "main": "入口文件名",  // 模块入口文件
  "viewEngine": "ejs/handlebars/pug/mustache",  // 视图引擎
  "language": "javascript",  // 开发语言
  "permissions": ["read", "write"],  // 权限列表
  "dependencies": [],  // 依赖列表
  "routes": [  // 路由配置
    {
      "method": "GET",
      "path": "/",
      "handler": "index"
    }
  ]
}
```

## UI 模块开发

### 1. 创建 UI 模块目录结构

```
./module/my-ui-module/
├── setting.json          # 模块配置文件
├── index.js              # 模块入口文件
├── views/                # 视图目录
│   ├── index.ejs         # 首页视图
│   └── ...
├── lib/                  # 模块库文件 (可选)
└── assets/               # 资源文件 (可选)
    ├── css/
    ├── js/
    └── images/
```

### 2. 配置文件示例

```json
{
  "name": "我的 UI 模块",
  "version": "1.0.0",
  "description": "这是一个示例 UI 模块",
  "intro": "这是一个示例 UI 模块，用于演示模块系统功能。",
  "author": "开发者名称",
  "page": true,
  "main": "index.js",
  "viewEngine": "ejs",
  "language": "javascript",
  "permissions": ["read"],
  "dependencies": [],
  "routes": [
    {
      "method": "GET",
      "path": "/",
      "handler": "index"
    }
  ]
}
```

### 3. 模块入口文件示例

```javascript
/**
 * 模块入口文件
 */

module.exports = {
  /**
   * 模块初始化函数
   * @param {Object} sandbox - 沙盒环境
   * @param {Object} config - 模块配置
   */
  init: function(sandbox, config) {
    console.log(`模块 ${config.name} v${config.version} 已初始化`);
  },

  /**
   * 模块 API 接口
   * @param {String} method - HTTP 方法
   * @param {String} path - 请求路径
   * @param {Object} params - 请求参数
   * @returns {Object} API 响应
   */
  api: function(method, path, params) {
    return {
      success: true,
      data: {
        message: '模块 API 响应',
        module: config.name,
        version: config.version
      }
    };
  },

  /**
   * 路由处理函数
   */
  handlers: {
    /**
     * 首页处理函数
     */
    index: async function(req, res) {
      return {
        title: '我的模块',
        content: '欢迎使用我的模块',
        timestamp: new Date().toISOString()
      };
    }
  }
};
```

### 3. API 模块入口文件示例

```javascript
module.exports = {
  init: function(sandbox, config) {
    console.log(`API 模块 ${config.name} 已初始化`);
  },

  api: function(method, path, params) {
    if (method === 'GET' && path === '/data') {
      return {
        success: true,
        data: { items: ['item1', 'item2', 'item3'] }
      };
    }
    
    return { success: false, message: 'Unknown endpoint' };
  }
};
```

## 视图引擎

### 支持的视图引擎

1. **EJS** (默认)
   - 扩展名：`.ejs`
   - 语法：`<%= variable %>`

2. **Handlebars**
   - 扩展名：`.hbs`
   - 语法：`{{variable}}`

3. **Pug**
   - 扩展名：`.pug`
   - 语法：缩进式

4. **Mustache**
   - 扩展名：`.mustache`
   - 语法：`{{variable}}`

### 视图引擎配置

在 `setting.json` 中指定：

```json
{
  "viewEngine": "handlebars"
}
```

### 安装模块

1. 将模块文件夹放入 `./module` 目录
2. 确保 `setting.json` 配置正确
3. 重启服务器使模块生效

### 卸载模块

1. 从 `./module` 目录删除模块文件夹
2. 重启服务器

### 更新模块

1. 替换模块文件
2. 更新 `setting.json` 中的版本号
3. 重启服务器

## 常见问题

### Q: 模块无法加载？
A: 检查以下几点：
- `setting.json` 文件格式是否正确
- `main` 字段指向的文件是否存在
- 模块是否通过了沙盒安全检查

### Q: 视图无法渲染？
A: 确保：
- 视图文件存在于 `views` 目录
- `viewEngine` 配置正确
- 视图语法符合所选引擎规范

### Q: 如何访问数据库？
A: 使用系统提供的 API 接口，不要直接访问数据库文件。

### Q: 模块间如何通信？
A: 通过系统提供的消息总线或直接调用 API。

### Q: 如何实现模块评论功能？
A: 系统已内置评论功能，用户可以在模块列表页面点击评论按钮发表评论。

### Q: 如何查看模块收藏数？
A: 系统会自动显示每个模块的收藏数量在模块卡片上。
